wb_task_run

int wb_task_run (int window, string command [, int estimated_ms = 0])

Starts an asynchronous background task that executes command outside the UI thread. The function returns a task identifier, or 0 on failure.

If estimated_ms is greater than zero, progress notifications are generated while the task runs.

Notifications and polling

Task notifications are delivered to the window callback through $param1:

  • WBC_TASK_PROGRESS ($param2 = task id, $param3 = progress percentage)
  • WBC_TASK_COMPLETE ($param2 = task id, $param3 = exit code)
  • WBC_TASK_ERROR ($param2 = task id, $param3 = error/exit code)
  • WBC_TASK_CANCELLED ($param2 = task id, $param3 = exit code)

Use wb_task_poll() to query status and wb_task_cancel() to request cancellation.

Thread safety limits

Only the main UI thread may access WinBinder windows/controls. Background tasks must not call UI APIs directly; instead they signal completion or progress and let the window callback update the UI.
